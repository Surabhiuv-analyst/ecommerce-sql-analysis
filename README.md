E-Commerce Sales Analysis (SQL Portfolio Project)

## Overview:
This project is an end-to-end SQL analysis of the Kaggle Superstore dataset, simulating a US-based e-commerce and retail company selling Furniture, Office Supplies, and Technology across 4 regions. The dataset contains 9,994 rows of real-looking sales transactions from 2014 to 2017, across 17 columns covering sales, profit, discount, shipping, and customer information.

The goal of this project was to practise writing structured SQL queries to answer real business questions — moving from basic data retrieval through to business insight generation, in the same way a data analyst would explore a new dataset on the job.

## Tools used:
SQL (SQLite, via DB Browser for SQLite)
Dataset: Kaggle Superstore dataset (9,994 rows, 17 columns)

## What each level covers:


Level 1 — Basic SELECT queries
Querying the raw dataset to understand its structure — selecting specific columns, viewing distinct values, and getting familiar with the data before deeper analysis.

Level 2 — Filtering and sorting
Using WHERE, ORDER BY, and comparison operators to narrow down and rank records — for example, filtering by region, category, or date range, and sorting by sales or profit.

Level 3 — Aggregation(COUNT, SUM, AVG, MIN, MAX)

Level 4 — Business insight queries
Writing queries that answer real business questions — for example, which product categories are most profitable, which regions are underperforming, and how discounts affect profit margins.

Level 5 — Portfolio-level analysis (in progress)
Bringing together multiple queries into a cohesive analysis with a clear narrative — identifying key findings and presenting them as a business-ready summary

## Key questions explored:


Which product categories generate the most revenue and profit?
Which regions are underperforming, and why?
How does discounting affect profit margins?
Which customer segments are most valuable?
